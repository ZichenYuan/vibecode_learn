# Note Template

This is the reference template for learning notes generated by the learn-mode skill. Every note saved to `~/learning-notes/` should follow this format.

---

## Template

```markdown
# {Topic Title}

**Date:** {YYYY-MM-DD}
**Context:** {What task triggered this learning — 1 sentence}
**Category:** {go | python | typescript | kubernetes | git | bash | patterns | etc.}

---

## What Is It?

{1-2 paragraph plain-English explanation of the concept. Write as if explaining to a smart friend who hasn't encountered this before. Use analogies where helpful.}

## Code Example

{Minimal, runnable code example demonstrating the concept. Include comments explaining key parts.}

```{language}
// Example code here
```

## Why It Matters

{When would you use this? What problem does it solve? Why should you care?}

- {Practical reason 1}
- {Practical reason 2}

## ⚠️ Common Gotchas

{1-2 common mistakes or pitfalls related to this concept.}

1. **{Gotcha title}** — {Explanation and how to avoid it}
2. **{Gotcha title}** — {Explanation and how to avoid it}

## ✅ Key Takeaways

{2-3 bullet points — the "remember these" summary}

- {Takeaway 1}
- {Takeaway 2}
- {Takeaway 3}

## 🔗 Related

- [{Related note title}]({relative-path-to-note.md})
- [{Official docs link title}]({url})

---

*Generated by learn-mode skill*
```

---

## Guidelines for Using This Template

### Length
- Aim for **50-150 lines** per note
- Be concise — this is a reference, not a textbook
- If a topic is large, split into focused sub-notes

### Code Examples
- Keep examples **minimal** — shortest code that demonstrates the concept
- Always include **comments** on non-obvious lines
- Show **both wrong and right** approaches when relevant (use ❌ and ✅)
- Use the actual language from the session (don't translate to a different language)

### Linking
- Link to related notes using relative paths: `[Error Handling](../go/error-handling.md)`
- Link to official docs when available
- After accumulating several notes, cross-reference them

### Updating Existing Notes
When appending to an existing note, add a new section:

```markdown
---

## Update: {YYYY-MM-DD}

**Context:** {What new thing was learned}

{New content following the same structure}
```

This preserves the learning timeline while keeping everything in one place.
